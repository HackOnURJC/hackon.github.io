Descripción del taller

En este taller se explicará en qué consiste una vulnerabilidad de buffer overflow y las consecuencias que puede tener para un sistema que se explote dicha vulnerabilidad. Además, se mostrará cómo pueden ser explotadas estas vulnerabilidades y, sobre todo, cómo debemos actuar como desarrolladores para intentar evitarlas.

Duración aproximada: 90-120 minutos.

Número máximo de plazas: 60.

Requisitos para participar en el taller: Sería deseable tener conocimientos básicos de programación en C y algo de arquitectura (entender qué es una dirección de memoria y cómo funciona, número de bits que ocupa un tipo de dato en memoria…). Si es posible, un ordenador Windows con una máquina virtual Linux instalada (o un ordenador con Windows o Linux, aunque sería deseable una máquina con los dos para mostrar técnicas en ambos sistemas).
